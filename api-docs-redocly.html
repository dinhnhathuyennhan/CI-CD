Tạo ticket hỗ trợ mới

Mở lại ticket đã đóng

Mở lại một ticket đã đóng. Chỉ thực hiện được khi ticket đang ở trạng thái đóng (trangthai = 2) và thời gian đóng trong vòng 2 tháng gần nhất. Nếu không truyền descriptions, hệ thống tự điền "MỞ LẠI CÂU HỎI".

Authorizations:
bearerAuth
path Parameters
user_id
required
string
Example: 23

UID của khách hàng. Phải trùng với user_id trong JWT token — hệ thống validate ở middleware.

id
required
string
Example: 456

ID của ticket cần thao tác

header Parameters
X-Request-Id
string
Example: req-abc123

Trace ID để debug request — optional, xuất hiện ở toàn bộ APIs trong hệ thống.

Request Body schema: multipart/form-data
optional
description
string <= 4000 characters

Mô tả lý do mở lại ticket, tối đa 4000 ký tự. Nếu không cung cấp, hệ thống mặc định sẽ là "MỞ LẠI CÂU HỎI"

Array of objects (FileAttachment) <= 3 items

Danh sách file đính kèm hỗ trợ cho việc mở lại ticket. Tối đa 3 file, mỗi file tối đa 10MB.

Responses
